# Build Guide

## Prerequisites

- ESP-IDF v5.5 installed and sourced (see [SETUP_GUIDE.md](SETUP_GUIDE.md))
- Python 3.11+

---

## What This Project Builds

This repository builds the **BYUI Badge Factory Loader** — the firmware
that lives permanently in the `factory` partition.  It is never
overwritten by OTA updates.

Boot behaviour:

- **Normal power-on (no buttons):** launches the installed student app
  within ~150 ms.  If no student app is installed, the loader UI opens
  automatically.
- **Hold A + B at power-on (~150 ms):** opens the factory loader UI
  (splash screen → WiFi config portal if needed → loader menu).

---

## Build

```bash
# Set the target chip (only needed once; result saved in sdkconfig)
idf.py set-target esp32s3

# (Optional) open the configuration menu
idf.py menuconfig

# Build
idf.py build
```

Build output:

| File                                        | Flash address | Description                                  |
|---------------------------------------------|---------------|----------------------------------------------|
| `build/bootloader/bootloader.bin`           | `0x0000`      | 2nd-stage bootloader                         |
| `build/partition_table/partition-table.bin` | `0x8000`      | Partition table                              |
| `build/ota_data_initial.bin`                | `0xf000`      | Blank OTA metadata (required on first flash) |
| `build/ebadge_app.bin`                      | `0x20000`     | Factory loader application                   |

Current binary size: **~1.1 MB** (87% of the 1.25 MB factory partition, 13% free).

## Flash the Badge

### First-Time Flash (clean badge)

Erase everything, then write all four binaries:

```bash
# Linux / WSL
idf.py -p /dev/ttyUSB0 erase-flash
idf.py -p /dev/ttyUSB0 flash

# Windows (ESP-IDF PowerShell)
idf.py -p COM3 erase-flash
idf.py -p COM3 flash
```

`idf.py flash` writes bootloader + partition table + ota_data_initial +
factory app in one step using the addresses above.

### Full Manual esptool Command

Generated by the build system — use this when `idf.py flash` is not
available:

```bash
python -m esptool --chip esp32s3 -b 460800 \
    --before default_reset --after hard_reset \
    write_flash \
    --flash_mode dio --flash_size 4MB --flash_freq 80m \
    0x0000  build/bootloader/bootloader.bin \
    0x8000  build/partition_table/partition-table.bin \
    0xf000  build/ota_data_initial.bin \
    0x20000 build/ebadge_app.bin
```

### Re-flash Factory Loader Only (preserving student app)

If the student-app OTA slots and `user_data` NVS should be left intact,
flash only the factory binary:

```bash
python -m esptool --chip esp32s3 -b 460800 \
    --before default_reset --after hard_reset \
    write_flash 0x20000 build/ebadge_app.bin
```

### Flash + Monitor

```bash
idf.py -p /dev/ttyUSB0 flash monitor
```

> Press **Ctrl+]** to exit the monitor.

## Entering ROM Download Mode (if idf.py flash fails)

1. Hold **IO0 (BOOT)** button.
2. Press and release **RESET**.
3. Release **IO0**.
4. Run the flash command above.

## Component Structure

The factory loader is composed of these components:

| Component       | Location         | Role                                       |
|-----------------|------------------|--------------------------------------------|
| `main`          | `main/`          | Boot decision; A+B check; loader init      |
| `buttons`       | `buttons/`       | GPIO driver for all 6 buttons              |
| `loader_menu`   | `loader_menu/`   | Interactive 5-item loader menu             |
| `portal_mode`   | `portal_mode/`   | SoftAP + QR-code WiFi config portal        |
| `ota_manager`   | `ota_manager/`   | OTA download from manifest URL             |
| `display`       | `display/`       | ILI9341 TFT driver (320×240)               |
| `leds`          | `leds/`          | WS2813B addressable LED chain              |
| `splash_screen` | `splash_screen/` | BYUI logo animation                        |
| `wifi_config`   | `wifi_config/`   | NVS storage for credentials + manifest URL |

To add a new component:

1. Create `<component_name>/CMakeLists.txt` and source files.
2. Add the directory to `EXTRA_COMPONENT_DIRS` in the top-level
   `CMakeLists.txt`.
3. Add the component name to `REQUIRES` in whichever component needs it.

## Building Student Apps (OTA Slots)

Student apps are **separate ESP-IDF projects** that flash into `ota_0`
or `ota_1`.  The factory loader installs them at runtime via the
loader menu — no manual flashing required in normal classroom use.

To build and manually flash a student app for development:

```bash
cd path/to/student_app
idf.py set-target esp32s3
idf.py build

# Flash to ota_0 (address 0x160000)
python -m esptool --chip esp32s3 -b 460800 \
    --before default_reset --after hard_reset \
    write_flash 0x160000 build/<student_app>.bin
```

After flashing, reset the badge **without holding A+B** — the factory
loader will detect the valid OTA image and boot into it immediately.

To return to the factory loader from a student app, the student app
should call:

```c
const esp_partition_t *factory =
    esp_partition_find_first(ESP_PARTITION_TYPE_APP,
                             ESP_PARTITION_SUBTYPE_APP_FACTORY, NULL);
esp_ota_set_boot_partition(factory);
esp_restart();
```

## AP IP Address (MAC-Derived, Per-Badge Unique)

Each badge computes its own SoftAP IP address from the last two bytes of
its factory eFuse MAC.

### Formula

Given MAC address `aa:bb:cc:dd:vv:ww`:

```text
AP IP = 192.168.(vv % 240).(ww % 240)
```

The last octet is clamped to a minimum of 1 to avoid `.0` network
addresses.  All badges share the SSID `BYUI_NameBadge` but have
different IPs, so they can operate in the same room without conflict.

### Example

| eFuse base MAC        | vv (byte 4)  | ww (byte 5)  | AP IP                    |
|-----------------------|--------------|--------------|--------------------------|
| `a0:b7:65:12:3c:f8`   | `0x3c` = 60  | `0xf8` = 248 | `192.168.60.8`           |
| `a0:b7:65:12:00:00`   | 0            | 0            | `192.168.0.1` (clamped)  |

### Finding Your Badge's IP

After flashing, open a serial monitor — the IP is logged at startup:

```
I (xxx) wifi_config: AP IP derived from MAC aa:bb:cc:dd:vv:ww → 192.168.X.Y
I (xxx) wifi_config: SoftAP started: SSID="BYUI_NameBadge"  IP=192.168.X.Y
```

The badge also displays the URL on-screen below the QR code.

### Reading the MAC Before Flashing

```bash
# Linux / WSL
esptool.py --port /dev/ttyUSB0 read_mac

# Windows ESP-IDF PowerShell
esptool.py --port COM3 read_mac
```

Output shows `MAC: aa:bb:cc:dd:vv:ww`. Apply the formula above.
